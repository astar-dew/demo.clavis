# CLAVIS 작업 기준 가이드

기준일: 2026-04-16  
적용 범위: `clavis` 폴더 작업 전반

## 1) 작업 범위 기준
- 기본 작업 대상은 `clavis` 폴더 내부 파일만 한다.
- 요청이 없는 폴더(`cafe24skin-study`, `totalfootball` 등)는 수정하지 않는다.
- `white`/`dark` 공통 반영 여부는 요청 조건에 따라 분기한다.

## 2) 문구/콘텐츠 작업 기준 (의료기기 + 주얼리)
- 의료기기 문구와 웰니스/주얼리 문구를 분리해서 다룬다.
- 질병 치료/개선으로 해석될 수 있는 단정 표현은 피한다.
- 절대/우월 표현(예: 유일, 1위, 최고, 100%)은 객관 근거가 없으면 축소/회피한다.
- 인증/허가 관련 문구는 실제 보유 문서 범위 내에서만 사용한다.

### 금지 또는 고위험 표현 예시
- 치료, 완치, 염증 완화, 혈액순환 개선(단정형), 통증 감소(단정형)
- 국가가 보증하는 안전한 테라피
- 국내 유일, 유일무이, 점유율 1위(근거 미기재)

### 권장 표현 방향
- 의료효능 단정 대신: 착용감, 디자인, 일상 웰니스, 사용자 경험 중심
- 단정 대신: “도움을 줄 수 있음”, “개인차가 있을 수 있음” 구조 활용
- 인증 표기는 사실 중심(인증명/적용 범위/기관)으로 최소화

## 3) 레이아웃 작업 기준
- 레이아웃 선택 UI는 기존 동작을 유지하되, 요청된 조건에만 분기 적용한다.
- 조건부 문구 변경은 `localStorage`의 `clavis-layout`, `clavis-theme` 값을 기준으로 처리한다.

## 4) 이번 작업에 실제 적용한 기준
- 요청: **layout3 white에만 심의 리스크 완화 문구 적용**
- 적용 파일: `asset/js/layout-switcher.js`
- 적용 조건:
  - 경로가 `/white/`
  - 테마가 `light`
  - 레이아웃이 `layout3`
- 조건 만족 시에만 문구를 완화하고, 그 외 상태에서는 원문으로 복원되도록 처리했다.

## 5) 검수 체크리스트
- [ ] 문구가 의료효능 단정으로 읽히지 않는가?
- [ ] 인증/허가 문구가 실제 근거와 일치하는가?
- [ ] `white`/`dark`, `layout1/2/3`에서 조건 분기가 의도대로 동작하는가?
- [ ] 요청 범위를 벗어난 파일 수정이 없는가?
